## 三、生财内部的 AI 员工体系

  

### 从 All in AI 到 All in Agent

  

**未来10年最大的机会是什么？只有一个——就是 Agent。**

  

Agent 不是说你在做什么 AI 工具，不是做什么其他行业、什么机会。Agent 会颠覆几乎所有的行业、所有的服务、所有的产品。

  

去年我们讲生财要 All in AI，但今年我们不是 All in AI 了，**我们是 All in Agent**。我们把 AI 更进化成叫 Agent，这是所有人都要去达成共识的一个点。

  

### 亦仁的 AI 员工军团

  

亦仁在生财内部自己搭建了几十个 AI 员工，全部进入生产线，现在生财的很多工作其实都是 AI 在做。在亦仁的带动下，整个生财团队内部，大家都在大量地创造各种 Agent。

  

这些 AI 员工全部在飞书左侧导航栏里排列，随时等亦仁给他们安排任务。这几十个 Agent 是在五六月份创建的，基于 Hermes + 飞书 CLI 的组合搭建了公司内部的 AI Agent 体系。

  

**Agent 的几大部门：**

  

**1. Agent 组织部（相当于人力部门）**

  

- **Agent 架构师**：拥有亦仁的所有权限，每天可以看所有人的聊天、群里面的发言、开的会、转的文档。架构师每天的任务就是告诉亦仁"昨天哪些工作流是可以 Agent 化的"。亦仁判断有价值，就说"OK，去创建一个 Agent 把这工作给替代了"。

  

- **Agent 运维官**：Agent 还是不稳定的，一会儿权限有问题，一会儿授权过期了。运维官可以自动定时去维护所有 Agent。

  

- **Agent 审核官**：现在公司里每天大概有十几个 Agent 上线，都需要审核。亦仁根本审不过来，所以专门做了审核官来审核权限是否 OK。

  

- **技术专家**：专门修 Agent。

  

**2. 内部运营类 Agent**

  

周报、申请、会议、财务、微信日报，已经完全可以做了。

  

**3. 内容生产类 Agent**

  

整个内容生产、选题、改写、发布、转入全部 Agent 化。

  

**4. 社群管理类 Agent**

  

精华帖筛选、打标签、用户洞察、航海群用户提出什么问题、最近用户有什么困惑，全部 Agent 化在做。

  

### 案例：多平台转录 Agent

  

亦仁不太喜欢看视频，长视频几十分钟或一两个小时，从来不看。小宇宙一两个小时的播客也听不完。YouTube 上三四个小时的长播客特别好，但根本没时间听。

  

市面上的转录工具要么摘要太过笼统，看不到细节；要么口水话太多。**细节才是拉开差距的地方。**

  

所以亦仁自己做了一个多平台转录 Agent，可以把主流视频平台、音频平台的内容转录成干净的文稿——既没有口水话，又能把细节呈现出来。

  

**这个 Agent 叫"小弟"**，看到任何好的帖子、博客、视频，直接转给他。比如 Claude 四个小时的教程，帮忙转录出来，然后在里面挑重点。而且因为设定了亦仁的身份和提示词，Agent 知道哪些内容对亦仁有帮助，**直接帮忙标注重点，相当于有一个特别懂你的超级助理**。

  

> 如果自己做不出来，市面上有人推出这个产品，一年一万块钱我是愿意付的。

  

### 人在 Agent 时代的角色

  

Agent 用得好之后，人基本上就是几个核心职能：

  

**1. 找到好问题**——你能不能知道你得干什么，能不能把问题定义清楚。**这是有想法的人的时代。**

  

**2. 做判断**——判断什么事情有价值，该往哪个方向走。现在基本上不是人在等 AI，是 AI 在等人，因为同时很多个进程在开发。你的判断力就非常重要。

  

**3. 定义标准**——你能不能知道真正好的东西是什么，把标准输出给 AI，AI 大概也能产出好东西给你。对我们应用层来讲，绝大多数时候 AI 只会给我们惊喜，我们基本上还没有实践到 AI 的边界。

  

**4. 高峰决策**——你还得去做最后兜底的人，不能完全交给 AI。

  

### Agent 与工作流的区别

  

去年工作流的概念比较火，但做工作流基本上还是围绕旧有的生产流程，把 AI 当成工具做中间部分的改造。**今年 Agent 出来之后，和工作流相比，整个效率是十倍好甚至百倍好的。**

  

关键区别：**AI 是个博士生，不是小学生。** 你应该完整地告诉他你的意图、目标、怎么实现，包括竞品样本，然后让 AI 去主导整个生产流程，人类做配合。这样需要判断和决策的点就会变少。

  

### 关于 AI 幻觉

  

AI 是人的能力的放大器。如果你自己做东西幻觉多，AI 的幻觉也多。如果你能够清晰地定义问题，AI 给你的产出是非常惊艳的。

  

如果 AI 产生幻觉，大概率是：

1. 你的 AI 模型还不太好，可能得换换模型

2. 你对 AI 的掌控能力还不够，AI 工程能力需要提升

3. 跟上下文工程有关系，给的上下文可能不够

  

**现在有个词很火叫 Harness——Agent 减掉大模型能力，剩下的都是 Harness。** 你有没有给他构建足够好的上下文，包括充分的授权、信任，让他理解你这个事情的目标、流程、结果。

## 四、统一技术栈：Codex + Hermes + 飞书 CLI

  

### 为什么要统一技术栈

  

在6月份之前，生财团队内部的技术栈非常杂——Claude Code、Codex、Hermes、Claude、Obsidian、飞书、企微、Windsurf 等等。

  

经过探索和实践，生财统一了技术栈为三件套：

  

**1. Codex（核心工作台）**

  

为什么统一到 Codex？不是不想用 Claude Code，实在是没办法——**封号太厉害了**。亦仁自己主力还在用 Claude Code，但基本上周二就把每周的额度用完了。团队每周可能都有几个号被封掉。所以 Claude Code 现在只能供给团队核心的几位工程师使用。

  

**Agent 工程师的配额**：200刀的 Codex（ChatGPT Pro）+ 250刀的 Claude Code Max 20X。要给工程师足额保障，才能产出更高质量的成果。

  

**2. Hermes（Agent 框架）**

  

如果你希望 Agent 能够自动维护、有记忆，甚至在飞书、企微、微信里面很好地去运行，就需要引入 Hermes 这个 Agent 框架。

  

小龙虾（OpenHands）在那个阶段完成了历史使命，带着整个人类往前跃了一步。现在推荐用 Hermes，非常稳定且好用。

  

**3. 飞书 CLI（协作平台）**

  

过去几年的工作都在飞书上，飞书开放了 CLI 之后，可以无缝地用 Agent 来操作飞书——发消息、看文档、看汇报、盯项目、整理内容。

  

### Obsidian 的定位

  

Obsidian 适合个体，本质上是一个知识库。但作为团队和公司来讲，更重要的是协作。协作在 Obsidian 里很难实现，但用飞书就非常顺畅。

  

**重要提醒：你只需要学一个东西——Codex。** 爱马仕和飞书 CLI 不用你学，直接让 Codex 去干就好了。包括几十个 Agent 的维护，亦仁都不维护的，都是让 Codex 去修。

  

### 关于知识库和 Skill 的新认知

  

亦仁最近的感受：对知识库和 Skill 这两个东西已经无感了。Codex 和 Claude Code 的进化在把这些东西慢慢内化。

  

现在 Codex 每次交互的内容都会默认沉淀成 Markdown 格式保存在本地知识库里。知识库基本上不会去看，是按照 Git 的方式做版本管理的。**AI 以自己最能理解的方式去管理知识库的内容——AI 懂就好了，人不用懂。**

## 五、产品方法论：言出法随

  

### Demo 文化替代评审文化

  

亦仁现在对公司的要求：**不要说有个想法来讨论一下，把你的 Demo 拿出来。** 你把一个真实的线上跑的产品拿出来给我看一下，行就行，不行就不行。

  

现在公司全员进后端。去年说全员学 AI 编程，今年6月份前说全员做 Agent，这周开了全公司会议说：**全员要进后端——能够调 API、打数据库、产品在域名上线、用 Git 方式去迭代产品。**

  

### Loop 的概念

  

现在有一个词非常火叫 **Loop**——在 AI 时代，个体的能力被极大丰富了，公司的部门分工在现在这个时代失效了。过去有想法要经过运营→产品经理→设计师→开发→测试，一轮又一轮。**现在如果你有想法，你自己就是一个 Loop，你自己就应该把产品直接开发出来并上线。**

  

全球最火的公司 Claude（Anthropic）就是这样——3000人的公司用项目制运作，有想法找两个人组队就开干。

  

**生财未来所有人都是 FDE，所有人都是全栈。** 现在公司好多运营都做出了生产级的产品。

  

### 没有开发、没有产品经理、没有设计师

  

没有业务方，因为没有人会给你需求了。现在所有人都在缺需求——有需求自己就实现了，不可能说一个需求还给别人帮忙做。**现在缺的是想法，有想法自己就做了。**

  

### 言出法随

  

**这是亦仁提出的 AI 时代创业方法论——当你有想法的那一刹那，这个产品就应该出来了。**

  

不是以前那种讨论、评审、出 PRD、出设计稿、找人开发，那都过时了。想法出来的那一刹那，Demo 就出来。亦仁现在基本上就是这样，想做一个产品，接下来可能20分钟就做出来了。

  

### 为什么老板要亲自做 Demo

  

**没有任何人比你更懂你的用户、你的需求、你的场景。** 包括你对长期的想象、使命、愿景，你的团队其实是不懂的。你把 Demo 做出来，所有人就立刻达成共识。

  

亦仁的核心职责就是非常快地把 Demo 做出来——**定义问题、定义审美、定义标准、定义方向**。做出来之后交给团队成员去持续迭代和维护。一天可能能定义十几个各种细分领域的 Demo。

  

### 快速迭代、允许屎山代码

  

做产品的方法论：

  

1. **快**——第一版不要管做得怎么样，先把最核心功能做出来推出去

2. **高频迭代**——亦仁一天大概提交几十次代码

3. **允许屎山代码**——不要一开始就追求完整架构，接受屎山代码

4. **到临界点再重构**——产品相对成熟、功能和需求相对清晰后再重构

  

> **"如果你的第一版产品上线你自己都不觉得尴尬，那说明你这个产品上线太晚了。"**

  

Token Rank 就是这样做出来的——花了20分钟做了个 Demo，都没注册域名，用了一个老域名的子域名直接上线，发到群里就有100多个圈友把 Token 统计发上来了。验证完了，才认真开始做这个产品。

  

重构成本现在非常低，一次 Codex 的5小时额度就够了。

**而且这个东西会开放 MCP 给大家。** 未来圈友也不是人去看的，用自己的 Agent 去看这些 Agent 的内容，筛选出真正需要的东西。

### 5. GitHub Talk

  

好的 Skill 和项目太多了，根本没法判断什么是好的。生财会做一个好的 Skill、好的项目仓库的认证和背书。

  

### 6. 数矿（数据集合）

  

很多人做 Agent 搞不定数据——小红书数据怎么拿、推特数据怎么拿、微信数据怎么拿。生财正在做一个帮你告诉各平台、各种数据怎么拿到的集合地，什么情况用爬虫、什么情况用接口、什么情况用 Cookie。

  

### 7. Agent 黑客松平台

  

很多企业想组织内部黑客松但缺方法流程。生财把组织黑客松的流程和方法变成产品。**未来的航海其实就是一个黑客松——所有人参赛，比赛谁把 Agent 做出来，好的 Agent 帮他分发、帮他售卖。**

  

### 8. 白话 AI 小百科

  

把常见的 AI 概念都展示出来——Loop、Harness、Token、神经网络等，配合案例，让不懂 AI 的人快速入门。

## 七、FDE——前端部署工程师

  

### 什么是 FDE

  

**FDE（Front Deployed Engineer，前端部署工程师）**——由 Palantir 提出的概念，派成员工去企业，借助 AI 强大能力把东西部署到客户那里。

  

现在不仅是 Palantir，OpenAI、Claude、国内大模型包括飞书都在招募和培养 FDE。

  

**亦仁认为 FDE 不仅仅是大厂大模型公司的机会，反而是我们这些个体、超级个体、普通人的机会。** 抓住 FDE 这个机会，也许能有十倍甚至百倍增长。

  

### 亦仁自己就在成为 FDE

  

过去几个月，亦仁正在把自己变成公司的最大 FDE。目标是公司所有的项目靠自己一个人都能交付到位。如果自己能做到，就可以把团队成员培养出来成为 FDE，然后一起服务更多企业。

  

**而且 FDE 不像外包，一个人可能只能服务两三个企业就到头了。随着 AI 模型能力发展，一个人能同时支撑几十家企业，价值几十倍放大。**

  

### FDE 需要什么能力

  

1. **AI 编程能力**——这是基础，必须能进入后端

2. **行业理解**——对行业的上下游、判断，只有在行业里服务了足够多的企业才能积累

3. **产品判断力**——知道什么产品有价值、什么不值得做

  

### 好的 FDE 会是谁

  

好的 FDE 会是那些行业里面本身做 2C 业务的企业主。比如做珠宝的老板，觉得珠宝未来没什么增量了，现在就在做 FDE，帮电商企业做 AI 化改造，做得津津有味。

  

**未来各行各业会有各行各业自己的 FDE，因为对行业的上下游理解是个很重要的事情。**

  

### FDE 的市场

  

FDE 现在日薪大概在2000到4000块之间，但供给非常少。**FDE 是一个至少十年的机会。** 7000万企业都需要做 AI 化转型，而真正会 AI 的人极其稀缺。

## 八、七条航海船

  

这次航海精心挑选，一共只有七条船（之前都是十几条），每一条都砍到最后不得不留下来的。

  

### 1. AI 做小程序

  

小程序成本极低、不需要供应链、能快速有反馈。在某书和某音还能享受推广红利，非常多的细分需求没有被充分满足。

  

**案例：股东超** —— 去年11月做虚拟产品小程序，第一个月十几万，后来月破百万。基本没有成本。

  

**案例：卧龙** —— 05年出生的小朋友，高中没毕业就辍学了，在某书上卖课堂管理小工具给老师，最好时一个月做了快50万。

  

> **在中国市场，再小的需求可能都有上千万人有这个需求。大胆去创造、大胆去做。**

  

### 2. AI 做海外网站

  

跟热词相关，比较快有反馈。仍然有很多圈友在做海外网站过程中拿到了很多反馈。

  

### 3. AI 做高质量视频

  

AI 做高质量视频、短电影、短剧的能力已经极大提升。教大家做有剧情的长 AI 视频，落地场景可以是短剧、带货、电影。

  

### 4. AI 视频带货

  

某视频号上 AI 视频带货非常火。一条自然流的数字人视频，点赞不到500，带大健康品居然卖了70万。还有楚川的弟弟，做专升本线索视频，两个月赚了27万。

  

### 5. 从零到一做 AI Agent

  

生财统一的技术栈：Codex + Hermes + 飞书 CLI，怎么搭出你的第一个 Agent，完整交给你。

  

### 6. AI 第二大脑

  

用 Obsidian 搭建知识库，以自媒体场景为主——选题、收集素材、生产素材、分发。

把你的业务经验、方法提炼成 Skill，共享给别人。以前写精华帖，现在做 Skill。

## 九、Token Rank 的实践心得

  

Token Rank 是亦仁真正从零到一做的生产级产品——包括所有的东西都是从零到一的，感受特别深刻。

  

### 开发过程

  

6月份做的时候，一天大概提交了将近40次代码。从凌晨5点一直提交到晚上11点，非常兴奋。经过这一次进后端、深入搞定了很多问题之后，能力比之前强了十倍。

  

### 当前数据

  

排行榜头部玩家一天大概消耗20亿左右 Token。

  

> **Token 消耗不一定代表你厉害，但是不消耗 Token 大概也是不厉害的。**

  

### 做完产品后的变化

  

1. **判断力更加重要**——多项目、多线程开发，在什么时候该做减法、什么时候该往前走，判断力就更重要了。推荐看《微信背后的产品观》。

  

2. **Loop 工程能力要上去**——能同时并发多项目多线程在跑。看了 OpenClaw 创始人 Peter，一个电脑开了将近40个线程，同时还把任务分发到更多电脑上，估计同时有100个线程在跑。

  

> **在极少的极聚焦的领域去做东西，但做得非常深非常透，产生的杠杆是乘法。**

## 十一、企业 AI 化改造

  

### 老板是第一责任人

  

**"老板不下场，没有好下场。"**

  

AI 真的是一把手工程。老板不知道 AI 的边界在哪里，团队里都不知道该怎么用。

  

这是局部最优解和全局最优解的对抗——让员工学 AI，最多组织效率提升30%，因为他只做了中间一个环节的优化。**但如果老板学了 AI，应该直接把原来的工作流扔掉，整个公司搭造一个全新的解构。**

  

### 改造步骤

  

**第一步：老板自己得懂。** 你能有 AI 编程能力，能通过 Codex 或 Claude Code 做出生产级项目。老板不懂，AI 改造第一步就失败了。

  

**第二步：鼓励团队百花齐放。** 也不要设定什么限制，提供支持就好——Token 支持、账号支持、网络支持。AI 账号可以报销，电脑升级也可以支持。这个阶段员工可能提效了，但多出来的时间不一定会去做业务，这是正常的。

  

**第三步：统一技术栈，树立标杆。** 把 Codex + Hermes + 飞书 CLI 统一，做培训。在内部聚合好的 Agent 和好的 Skill，建立评价体系，奖励优秀的同学。大家看到好的标杆，就会形成氛围和压力，所有人都会进一步学习。

  

这是一个**从上到下再从下到上**的过程：老板自己学、实践，了解边界和方向；鼓励团队去做；然后有了规范标准后开始大量涌现。

  

> 生财内部有个 AI 学习小组，每天都会分享自己是怎么用 AI 的、有什么新的玩法。已经开始涌现了。

  

### 对团队的要求

  

之前30天的工作，一个星期要干完；之前一个星期干的工作，一天要干完。

  

8月底要有人能做到稳定的10倍产出，同时有10个进程在产出。当所有人每天在生财都可以做到稳定的10个亿 Token 消耗的时候，一个人创造的价值相当于以前一个团队的总和。

## 十三、总结

  

### 未来的差距

  

**人跟人的差距肯定不是谁更努力了，而是谁能调度更多 Agent。**

  

有人能同时调度上百个 Agent，同时 Run 10个项目——这个差距太明显了。

  

### 最重要的建议

  

**如果你觉得上班影响你学 AI，那你应该辞职学 AI。** 只是这个 AI 会更具化成叫 Agent。

  

掌握 AI 的工程能力、掌握 AI 编程，这是所有职场人、所有老板的必备能力。**就像以前有人问"我要不要学上网"一样。你越早上，就能越早享到红利。**

  

> **最便宜的还是 Token，最稀缺的是好的想法和好的判断力。**

  

好的想法和好的判断力怎么来？**实践出真知。弄脏双手，动手，不断犯错、不断尝试、不断学习。**

  

如果只记住一句话，就是：**用好 Codex，就是目前唯一的路。**

  

### 进入正循环

  

每两三个月会感受到自己十倍的提升。两个月前觉得自己是个傻子，两个月后又觉得非常厉害，再过两三个月又是一种感觉。

  

**现在进入一个时代——你想要什么，就把它创造出来。言出法随。**
