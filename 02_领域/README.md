# 🔄 领域

您长期维护的持续性责任和活动范围。

## 目的

领域是**持续性责任**，它们：
- 没有结束日期
- 需要持续关注
- 定义您的角色和承诺
- 需要定期维护

## 这里应该放什么

### 个人领域
- 健康与健身
- 财务
- 人际关系
- 个人发展
- 家庭与环境

### 职业领域
- 团队管理
- 客户关系
- 专业技能
- 行业知识
- 人脉网络

### 创作领域
- 写作练习
- 内容创作
- 学习与研究
- 副业项目

## 领域结构

每个领域可能包含：
```
领域名称/
├── README.md          # 概览和原则
├── 目标/              # 当前关注领域
├── 资源/              # 参考资料
├── 回顾/              # 定期检查
└── 想法/              # 未来可能性
```

## Claude Code 工作流

### 领域回顾
```
回顾我的【领域名称】领域。
你在最近的笔记中看到什么模式？
哪些方面需要更多关注？
```

### 设定目标
```
帮我为【领域】设定季度目标。
根据最近的活动，我应该优先考虑什么？
```

### 发现空白
```
分析我的【领域】笔记。
有哪些重要主题我没有追踪？
我应该问哪些问题？
```

### 跨领域分析
```
比较我的【领域A】和【领域B】笔记。
它们在哪里重叠？
它们如何相互启发？
```

## 领域 vs 项目

**问问自己**：这有结束日期吗？

| 领域（持续性） | 项目（有时限） |
|---------------|---------------------|
| "健康" | "6月前减重10磅" |
| "财务" | "制定2024年预算" |
| "写作" | "完成博客系列文章" |
| "学习" | "完成Python课程" |
| "客户" | "交付X项目" |

## 维护实践

### 每周
- 快速扫描每个领域
- 记录任何紧急需求
- 捕捉新想法

### 每月
- 回顾目标和进展
- 更新资源列表
- 清理旧笔记

### 每季度
- 使用 Claude Code 深度回顾
- 调整关注领域
- 归档过时材料

## Claude Code 提示词

### 健康检查
```
对我所有的领域做一次健康检查。
哪些有最近的活动？
哪些被忽视了？
```

### 平衡评估
```
查看我的领域，我在哪里投入最多精力？
什么失衡了？
```

### 整合机会
```
找出我不同领域之间的联系。
哪里可以通过更好的整合创造价值？
```

## 提示

- **领域催生项目** - 项目往往从领域需求中产生
- **保持活跃** - 不活跃的领域应该归档
- **定期回顾** - 没有关注的领域会漂移
- **记录标准** - "好"是什么样的？
- **追踪指标** - 即使是定性领域也能从度量中受益

## 记住

领域是您生活的背景。它们不会完成，但会演变。使用 Claude Code 帮助您看到模式并保持所有责任之间的平衡。
