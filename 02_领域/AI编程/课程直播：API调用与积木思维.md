### 课程核心观点与总结

本堂课的核心目标是为AI产品开发的初学者**祛魅和赋能**，指出构建一个AI应用并非遥不可及的“抽卡”魔法，而是一个有章可循、以创意为核心的“搭积木”工程。

老师首先提出了一个核心框架：**AI产品 = 前端交互 +** **API****能力**。基于此，他将AI产品划分为三个层次：**简单产品**（单一API，竞争激烈，比拼的是创意和先发优势，如早期的ChatPDF）、**复杂产品**（多API串联，是独立开发者和初创团队的蓝海，更容易成功）和**顶级产品**（拥有自研的、独占的API，如OpenAI和Midjourney）。其核心论点是，对于大多数开发者而言，真正的机会在于**“复杂的AI产品”**，通过创造性地组合、串联市面上已有的各种API，像搭积木一样，解决特定的用户问题，从而创造出独特的价值。这其中，**“用积木的眼光去看待API”**是贯穿始终的关键思想。

为了解决新手在实践中最大的痛点——调用API时反复失败、如同“抽卡一整个通宵”的挫败感，老师给出了一个极其重要的实践方法论：**“API调用四要素”**。他强调，在编写任何代码之前，必须通过Postman等工具，提前验证并集齐 **API Key、示例代码、示例输入和至关重要的示例输出**。只要将这四要素完整地提供给AI编程助手（如Cursor），就能实现“一把出”，极大地提升开发效率和确定性。

整堂课通过一个从0到1构建“互联网黑话翻译器”的生动演示，将上述理论付诸实践。从最初的单一API功能实现，到串联第二个TTS（文本转语音）API，再到最后用Cloud Code一键美化UI，完美地诠释了“先功能验证、再串联扩展、后美化迭代”的敏捷开发流程。这不仅证明了“四要素”方法的有效性，也展示了现代AI工具链（Postman + AI编程助手）的强大威力，最终向学生传达了一个鼓舞人心的信息：**技术的门槛正在被AI工具抹平，成功的关键回归到了发现真实需求和发挥创意的能力上。**

## 课程详细笔记

1. AI产品的本质：用积木的眼光看待API
    

AI产品最根本的构成可以简化为以下公式：

Generated code

```Plain
      AI产品 = 前端交互体验 + API能力
```

#### 三种AI产品类型

|   |   |   |
|---|---|---|
|类型|定义|特点与案例|
|简单的AI产品|简单的交互 + 单一API|- 核心竞争力：创意和获取用户的效率。技术门槛低，同质化严重。<br>- 案例：ChatPDF（第一个想到此创意）、Notion AI（利用庞大用户基数）、Grammarly。|
|复杂的AI产品|个性化交互 + 多个API串联|- 核心竞争力：搭积木的创意。通过Workflow、Agent、硬编码等方式将多个API组合，创造性地解决问题。<br>- 更容易成功，因为能有效避免同质化竞争。<br>- 案例：黑井视频翻译（早期版本是语音识别+大模型翻译+TTS+口型同步等多个API的组合）、Manless。|
|顶级的AI产品|独有的前端 + 自研私有的API|- 核心竞争力：拥有别人没有的、自研的底层API能力，构成了坚实的技术壁垒。<br>- 案例：GPT、Midjourney、Suno。|

**核心主张**：对于初学者和小型团队，应该将重心放在“复杂的AI产品”上，多思考如何串联API来创造新价值。

2. 如何告别“抽卡式”开发：API调用的四要素
    

很多新手在调用API时反复失败，过程痛苦且低效，如同“抽卡”。这完全可以避免。

> **核心方法**：在你写代码之前，通过工具（如Postman）提前验证API，并**集齐以下四个要素**，再交给AI编程助手，就能大概率“一把出”（一次成功）。

#### API调用四要素

1. **能调通的API Key**：也叫令牌（Token）或Secret，是访问API的凭证。
    
2. **API调用的示例代码**：通常在API文档中提供，`cURL`格式最为通用和推荐。
    
3. **示例输入 (Request Body)**：调用API时发送给服务器的具体数据，比如要翻译的文本、要生成的图片描述等。
    
4. **示例输出 (Response Body)**：**这是最关键也最容易被忽略的一点**。它是API成功调用后，服务器返回的数据格式和内容。提前获取它，AI编程助手才能精确地知道如何解析返回结果。
    
5. 实战演练：从0到1构建“互联网黑话”应用
    

#### 步骤一：使用 Postman 调通 API 并集齐四要素

1. **目标**：调用大模型API，实现普通话到互联网黑话的翻译。
    
2. **工具**：`Postman` (Web版: `web.postman.co`)。
    
3. **流程**：
    
    1. 在API提供商网站（如`OpenRouter`）找到目标API，获取`API Key`。
        
    2. 复制该API的 `cURL` 示例代码。
        
    3. 打开Postman，点击 `Import`，将复制的`cURL`命令粘贴进去。
        
    4. 在Postman的请求设置中（通常是`Headers`或`Authorization`），将占位的Key替换成你自己的`API Key`。
        
    5. 在`Body`中，修改**示例输入**，改成你想测试的内容。
        
    6. 点击 `Send` 发送请求。
        
    7. 如果下方返回 `Status: 200 OK`，说明API调用成功。**复制下方返回的JSON内容，这就是宝贵的“示例输出”**。
        
4. **产出**：此时，你已成功集齐“API调用四要素”。
    

#### 步骤二：使用 AI 编程助手（V0 / Cursor）进行编码

1. **目标**：创建一个Web界面，实现黑话翻译功能。
    
2. **工具**：`V0` 或 `Cursor`。
    
3. **流程**：
    
    1. 向AI编程助手下达明确指令，描述产品功能（用户输入普通话，点击按钮，输出黑话）。
        
    2. **将上一步集齐的“四要素”完整地粘贴给AI**，告诉它这就是需要调用的API。
        
    3. AI助手会根据你的需求和API信息，一次性生成前端和后端代码。
        

#### 步骤三：串联多个 API（增加 TTS 语音功能）

1. **原则**：**一个一个地加，验证一个，再加下一个**。不要试图一次性让AI实现所有复杂功能。
    
2. **流程**：
    
    1. 在`faul.ai`等平台找到TTS（文本转语音）API。
        
    2. **重复步骤一**：用Postman调通TTS API，获取其对应的“四要素”。（注意：某些异步API可能需要调用两次，一次提交任务，一次查询结果）。
        
    3. **重复步骤二**：回到项目中，告诉AI助手：“在现有基础上，增加一个功能。把翻译后的黑话文本，调用下面这个新的TTS API，生成音频并提供一个播放按钮。”
        
    4. 将TTS API的“四要素”提供给AI。
        
    5. AI会自动修改代码，将两个API串联起来。
        

#### 步骤四：使用 AI 编程助手（Claude Code）一键美化UI

1. **目标**：将功能验证通过但简陋的界面，快速换成专业美观的模板。
    
2. **工具**：`Claude Code`（或其他AI编程工具）。
    
3. **流程**：
    
    1. 向AI下达指令，如：“我觉得我的界面太丑了，请帮我套用这个专业的UI模板，并增加Header、Footer和FAQ板块。”
        
    2. AI会自动分析现有代码结构和你的新需求，生成全新的、专业的UI界面代码。
        
4. 关键工具与技巧总结
    

- **Postman**：在写代码前，**必须使用它来验证API**，这是保证“一把出”的关键。
    
- **AI 编程助手对比**：
    
    - `Cursor`：**推荐新手使用**。像一个结对编程的伙伴，能看到详细日志，便于调试和“干中学”。
        
    - `V0` / `Vellum`：适合快速生成UI原型，但调试时像个“黑盒”，不透明。
        
    - `Claude Code`：像一个“实习生”，适合执行边界清晰的、完整的任务，交互较少。
        
- **API Key管理**：开发测试时使用的Key，在直播或分享后要**立即删除**，养成良好安全习惯。
    
- **迭代思路**：**先功能，后UI**。先用最简单的界面实现核心功能（MVP），验证通过后，再用AI工具快速美化。
    

5. Q&A 精选
    

- **问：多个平台的API能串联吗？**
    
    - **答**：完全可以。本次演示就串联了`OpenRouter`和`faul.ai`两个不同平台的API，方法完全一样。
        
- **问：调用API总是超时怎么办？**
    
    - **答**：这是Next.js等框架的默认设置问题。你需要告诉AI编程助手：“请帮我修改Next.js的超时时间限制”，或者自己去配置文件中修改。这需要你补充一些`Next.js`的内功。
        
- **问：如何验证一个产品想法（卖空气）？**
    
    - **答**：做一个精美的落地页（Landing Page），详细描述你的产品功能，然后放一个**假的付费按钮**。统计点击该按钮的用户比例，甚至可以在点击后弹窗收集意向用户的邮箱。这样在开发前就能低成本地验证市场需求。
        
- **问：学习一个月还没出单，很焦虑怎么办？**
    
    - **答**：这是正常现象。1000个学员中可能只有1%的人能快速出单，这通常与他们之前的积累有关。放平心态，持续学习和实践。
        
- **问：为什么不给示例输出，AI就容易出错？**
    
    - **答**：如果你不提供示例输出，AI在调用API后，不知道服务器会返回什么格式的数据，它只能去“猜”。这大大增加了不确定性。提供了示例输出，不确定性就降为0，它就能精确地编写数据解析代码。
