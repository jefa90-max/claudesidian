## 二、统一写作模板

每章根据内容需要使用以下结构，不强制补齐不适用的小节：

1. **这一章解决什么问题**：用一段生活化场景说明读者为什么需要这一步。
2. **先看连接关系**：涉及多台设备时，先给一张简单文本图。
3. **开始前准备**：只列本章真正需要的设备、账号和信息。
4. **具体操作**：每一步都写明在哪台设备、哪个软件或哪个设置页面操作。
5. **操作结果**：紧跟在关键步骤后说明界面上会出现什么，不另设“本章验收清单”。
6. **遇到问题时先检查什么**：只写本章最常见的两到四个问题。

统一语言规则：

- 先说人话，再给 SSH、CLI、端口转发等术语；
- 一句话只完成一个动作；
- 图形界面路径使用“设置 → 页面 → 按钮”；
- 命令必须标明在 Windows、Mac mini 还是手机中执行；
- 不写 Docker、Git、Node.js、数据库、开发框架或编程方法；
- 不写 Claude Code 与 Codex 的功能分工；
- Claude Code 始终描述为 Mac mini 上的 CLI；
- 电脑使用 Claude Code 的链路固定为“VS Code Remote SSH → Mac mini 终端 → Claude Code CLI”；
- 手机使用 Claude Code 的链路固定为“Tailscale → Termius SSH → Mac mini → tmux → Claude Code CLI”；
- Codex 的链路固定为“Codex 桌面端或手机 App → Codex 远程连接 → Mac mini”。

## 三、事实核验规则

开始写安装步骤前，逐项查看当前官方文档，不直接照搬原手册中的旧界面或安装命令：

- Apple：Remote Login、睡眠、电源、防火墙和 Time Machine；
- Tailscale：macOS、Windows、手机安装，MagicDNS 与 key expiry；
- VS Code：Remote SSH 与端口转发；
- ToDesk、网易 UU：macOS 被控端、Windows 控制端和系统权限；
- Termius：手机 SSH 主机配置；
- tmux：安装与会话恢复命令；
- Claude Code：CLI 安装和远程终端使用；
- OpenAI Codex：桌面端、手机 App 与远程连接；
- 微信开发者工具：预览二维码和真机调试。

若官方文档与已经跑通的个人流程不一致，正文同时标注：

- “官方通用步骤”；
- “本文实际采用的步骤”；
- 差异产生在哪台设备或哪个网络环节。
