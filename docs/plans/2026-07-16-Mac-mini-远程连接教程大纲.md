# 把 Mac mini 变成随时在线的远程工作主机

> 副标题：出差不断线、多设备连接、远程使用 AI 工具与微信真机调试完整教程

## 一、教程定位

这是一篇准备发布到飞书的单篇长教程。文档前半部分帮助读者理解背景、架构与工具职责，后半部分提供可以按目录跳转的安装和操作步骤。

教程同时面向两类读者：

1. **非技术创作者、产品经理和个体开发者**：会使用电脑和 AI 对话，但不熟悉终端、SSH、端口转发等概念。
2. **AI 编程初学者**：希望从不同设备连接同一台 Mac mini，并持续使用远程服务与 AI 工具。

正文按零基础读者能够理解的方式编写。必要的技术原理放在提示框中，不在主线中展开同类产品横评或高级工程化内容。

## 二、真实背景与核心问题

教程从以下四个真实问题切入：

1. 以前使用笔记本开发，但经常出差；路上网络中断、电脑合盖或没电后，AI 与开发任务无法继续执行。
2. 笔记本性能有限，同时运行 AI 工具和多个服务时容易卡顿。
3. 使用多台电脑时，需要反复同步文件、软件环境和运行状态。
4. 除固定主机外，希望其他电脑和手机也能连接 Mac mini，并查看尚未正式部署的 Web 或小程序。

教程需要传达的核心观念是：

> 不是为了远程控制一台 Mac mini 而堆砌工具，而是把需要持续运行的软件与服务集中到一台固定主机。其他设备只负责连接、操作和预览。

## 三、教程完成标准

读者完成主线后应当能够：

- 从外地使用 Windows 笔记本安全连接 Mac mini；
- 通过 VS Code Remote SSH 使用 Mac mini 上的文件和终端；
- 使用远程桌面操作微信开发者工具等图形界面软件；
- 让远程服务和 CLI 任务在断网、合盖后继续运行；
- 通过 VS Code Remote SSH 终端使用 Mac mini 上的 Claude Code CLI；
- 通过 Codex 桌面端读取 SSH config 并连接 Mac mini；
- 在手机上通过 Tailscale 和 Termius SSH 查看并继续 Claude Code CLI 会话；
- 在手机 App 的 Remote 中进入已配对的桌面主机和 Mac mini 远程环境；
- 更换电脑后继续连接同一台 Mac mini；
- 在其他电脑或手机上访问 Mac mini 运行的 Web 服务；
- 通过笔记本转发端口，在手机上完成微信小程序真机预览；
- 理解最基本的远程访问安全与备份边界。

## 四、整套工作流

### 4.1 设备分工

```text
Mac mini：持续在线的远程主机
├─ 保存文件与远程工作环境
├─ 运行 Web、API 等本地服务
├─ 运行 Claude Code CLI 与长期任务
└─ 运行微信开发者工具

Windows 笔记本：远程控制台
├─ 通过 Tailscale 找到 Mac mini
├─ 通过 VS Code Remote SSH 使用文件、终端和 Claude Code CLI
├─ 通过 VS Code 转发开发端口
├─ 通过 ToDesk 或网易 UU 操作 Mac mini 桌面
└─ 通过 Codex 桌面端的 SSH Connection 使用 Mac mini

手机：移动连接与预览终端
├─ 通过 Tailscale 找到 Mac mini
├─ 通过 Termius SSH 查看并继续 Claude Code CLI 会话
├─ 通过手机 App 的 Remote 进入已配对的 Codex 主机
├─ 访问转发后的 Web 或 API
└─ 扫描微信开发者工具二维码进行真机调试
```

### 4.2 五类远程连接

| 通道 | 作用 | 使用工具 |
|---|---|---|
| 私网通道 | 让外地设备安全找到 Mac mini | Tailscale |
| 终端通道 | 在电脑上使用远程文件、终端、Claude Code CLI 和端口转发 | VS Code Remote SSH |
| 手机 SSH 通道 | 在手机上查看并继续 Claude Code CLI 会话 | Tailscale + Termius |
| 桌面通道 | 操作微信开发者工具等 GUI 软件 | ToDesk / 网易 UU |
| Codex 连接 | 桌面端通过 SSH 连接 Mac mini，手机端进入已配对主机 | Codex 桌面端 / 手机 App Remote |

### 4.3 微信小程序真机调试链路

```text
手机
→ 访问笔记本地址
→ 笔记本上的 VS Code 转发端口
→ Mac mini 上运行的本地 API

同时：
手机扫描 Mac mini 上微信开发者工具生成的二维码
→ 打开开发版本小程序
→ 调用经笔记本转发的本地 API
```

这一场景的重点不是“完全绕过微信预览流程”，而是“不必先把 API 和 Web 服务部署到正式服务器，就能完成外出环境下的真机调试”。

## 五、详细大纲

### 第一篇：先看懂这套工作流

#### 第 1 章　我为什么需要一台持续在线的远程工作主机

##### 1.1 笔记本开发遇到的四个问题

- 出差路上断网、合盖和没电导致任务中断；
- 性能有限，多项服务同时运行容易卡顿；
- 多台电脑之间同步文件、软件环境和运行状态很麻烦；
- 本地服务难以在其他电脑和手机上随时查看。

##### 1.2 从“随身开发”改成“固定主机计算”

- Mac mini 负责持续运行；
- 笔记本不再保存第二套运行环境；
- 手机和其他电脑作为入口，而不是新的工作主机。

##### 1.3 这套方案解决什么、不解决什么

- 解决环境统一、任务持续和多设备访问；
- 不等于云服务器，也不替代正式部署；
- 网络完全不可用时仍无法远程操作，但 Mac mini 上已经启动的任务可以继续执行。

#### 第 2 章　最终要搭成什么样

##### 2.1 一张图看懂设备之间的关系

- 展示 Mac mini、Windows 笔记本、手机和互联网之间的连接关系；
- 使用生活化比喻解释“主机、控制台和预览终端”。

##### 2.2 为什么远程工作环境只保留一份

- 避免多台设备的文件与软件状态不一致；
- 减少重复安装和同步；
- 更换客户端时无需重新配置 Mac mini。

##### 2.3 为什么要区分五类远程连接

- 私网通道解决“如何找到”；
- 电脑终端通道解决“如何使用文件、命令行与 CLI 工具”；
- 手机 SSH 通道解决“如何随时查看 Claude Code CLI 会话”；
- 桌面通道解决“如何操作 GUI 软件”。
- Codex 专用连接解决“如何从桌面端或手机 App 进入 Codex 远程会话”。

#### 第 4 章　你是否适合使用这套方案

##### 4.1 适合的人

- 经常出差或在多地点办公；
- 经常运行需要持续在线的 AI CLI 任务；
- 需要 macOS 或微信开发者工具；
- 希望电脑和手机随时访问同一台远程主机。

##### 4.2 可能不需要完整搭建的人

- 只在一台电脑上偶尔使用相关工具；
- 不需要长期任务、远程预览或图形界面软件；
- 无法为固定主机提供稳定电源和网络。

##### 4.3 搭建前必须接受的现实

- 固定主机需要保持开机和联网；
- 远程入口必须做好账号与密钥保护；
- 远程主机仍然需要备份；
- 第一次搭建需要完成多条连接通道，但之后更换设备会更轻松。

#### 第 5 章　搭建路线与完成目标

##### 5.1 第一阶段：能连接

- Tailscale 中能看到 Mac mini、Windows 和手机三台设备；
- Windows 能通过 SSH 登录 Mac mini；
- VS Code 能打开 Mac mini 上的远程目录和终端；
- 手机通过 Tailscale 和 Termius 能 SSH 登录 Mac mini。

##### 5.2 第二阶段：能持续工作

- 需要远程访问的服务在 Mac mini 上运行；
- Claude Code CLI 和终端任务放入 tmux；
- 笔记本断网重连后能恢复原会话；
- 手机能继续查看 Claude Code CLI 会话，并通过 Codex App 进入 Codex 远程连接。

##### 5.3 第三阶段：能跨设备预览

- 笔记本能访问 Mac mini 上的 Web；
- 手机能访问经过笔记本转发的 API；
- 手机能扫描开发者工具二维码并完成小程序真机调试。

### 第三篇：日常怎么用

#### 第 16 章　出差时建立远程连接

- 确认 Tailscale 在线；
- 使用 VS Code 连接 Mac mini；
- 恢复 tmux 中的 Claude Code CLI 或其他长期会话；
- 检查需要访问的远程服务；
- 需要 GUI 时再打开远程桌面；
- 只使用手机时，通过 Termius 或 Codex App 进入对应会话。

#### 第 17 章　网络中断后如何继续任务

- 区分“连接中断”和“Mac mini 上的任务停止”；
- 重新连接 Tailscale 与 SSH；
- 恢复 VS Code 工作区和 tmux 会话；
- 检查 Claude Code CLI 与远程服务状态。

#### 第 18 章　如何在电脑之间切换而不搬运行环境

- 新电脑只安装连接工具；
- 不在每台客户端复制一套运行环境；
- 继续连接同一台 Mac mini；
- 正确同步 SSH 与账号设置。

#### 第 19 章　在手机上继续 Claude Code 与 Codex 会话

- 先连接手机 Tailscale，再使用 Termius SSH 登录 Mac mini；
- 通过 tmux 恢复已经运行的 Claude Code CLI 会话；
- 在手机上查看 Claude 对话和任务状态；
- 使用手机 App 的 Remote 进入已配对的桌面主机；
- 明确两条路径互不替代，不把手机 App 的 Remote 写成通用 SSH 客户端。

#### 第 20 章　如何远程查看 Web 和小程序

- 查看 Web 时只需要终端通道和端口转发；
- 查看小程序时还需要桌面通道和开发者工具二维码；
- 用一张流程卡分别列出电脑预览与手机真机调试步骤。

#### 第 23 章　常见故障排查

按连接链路从外到内排查：

1. Mac mini 是否开机和联网；
2. Tailscale 是否在线；
3. SSH 是否能连接；
4. VS Code Remote SSH 是否正常；
5. 手机 Termius 是否通过 Tailscale 连接了正确主机；
6. tmux 中的 Claude Code CLI 会话是否还在；
7. Codex 桌面端或手机 App 的远程连接是否在线；
8. 目标服务是否监听正确端口；
9. VS Code 端口是否正确转发；
10. 手机访问的是否为笔记本地址；
11. 微信开发者工具是否生成了正确的预览版本。

#### 第 24 章　其他设备的替代接入方式

本章不做产品横评，只说明设备变化后的对应关系：

- Windows 客户端换成 macOS 后，整体架构不变；
- 手机系统变化不影响 Web 访问的基本链路；
- 如果项目不需要 GUI 软件，可以不安装远程桌面；
- 如果不需要在手机上使用 SSH，可以不安装 Termius；
- 如果只使用一种 AI 编程工具，可以省略另一种。
