# PARA Vault 目录重构计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 按照 PARA 方法论重构 Obsidian Vault 目录结构，修正 Areas/Projects/Resources 的分类错误，建立清晰的知识管理体系。

**Architecture:** 将错误归类为"领域"的内容按 PARA 标准拆分到正确的分类中：平台运营类移入项目、参考资料类移入资源、保留真正的持续责任领域。同时收编游离在 PARA 体系外的 Clippings 文件夹。

**核心原则:** Obsidian 默认使用「最短路径」解析 wikilink，移动文件只要不改文件名就不会断链。唯一的 wikilink 引用 `[[claude code批量运行skill]]` 在移动后仍可正常解析。
