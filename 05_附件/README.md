# 📎 附件

用于存储图片、PDF和其他非文本文件。

## 目的

集中存放:
- 图片和截图
- PDF和文档
- 电子表格和数据文件
- 音频和视频文件
- 笔记中引用的任何二进制文件

## 组织结构

```
05_附件/
├── 已整理/             # 已处理的文件,有良好的命名
│   ├── 图片/
│   ├── PDF文档/
│   └── 数据/
├── IMG_*.png          # 未处理的手机图片
├── Screenshot*.png    # 未处理的截图
├── CleanShot*.png    # 未处理的CleanShot文件
└── *.pdf             # 各种PDF文件
```

## 命名规范

### 处理前
- `IMG_1234.png` (来自手机)
- `Screenshot 2024-03-15 at 2.30.45 PM.png`
- `CleanShot 2024-03-15 at 14.30.45.png`
- `document(1).pdf`

### 处理后
- `2024-03-15_项目架构图.png`
- `2024-03-15_会议白板.jpg`
- `API文档_v2.pdf`
- `客户访谈记录.pdf`

## 辅助脚本

使用 `pnpm` 运行:

### 查看状态
- `attachments:list` - 列出未处理的文件
- `attachments:count` - 统计未处理文件数量
- `attachments:organized` - 统计已整理文件数量
- `attachments:sizes` - 显示最大的文件
- `attachments:recent` - 最近7天添加的文件

### 查找问题
- `attachments:orphans` - 未被任何地方引用的文件
- `attachments:refs [filename]` - 查找文件的引用

### 整理
- `attachments:create-organized` - 创建"已整理"文件夹

## 最佳实践

### 文件大小
- 图片保持在2MB以下以适配Git
- 压缩大型PDF
- 视频使用外部存储
- 提交前优化图片

### 命名
- 包含日期: `YYYY-MM-DD`
- 描述性但简洁
- 使用下划线而非空格
- 如相关,包含版本号

### 链接
```markdown
# 嵌入图片
![[05_附件/已整理/diagram.png]]

# 链接PDF
[[05_附件/已整理/document.pdf]]

# 带描述
![[05_附件/已整理/chart.png|销售图表Q1]]
```

## 处理工作流

1. **捕获**: 将文件保存到 `05_附件/`
2. **审查**: 查看内容,确定目的
3. **重命名**: 给予描述性的、带日期的名称
4. **整理**: 移动到 `已整理/` 子文件夹
5. **链接**: 更新笔记中的引用
6. **清理**: 删除孤立文件

## Claude Code 提示词

```

```

### 存储审计
```
分析附件存储:
- 总大小
- 最大文件
- 文件类型分布
- 孤立文件
```

## 提示

- **每周处理** - 不要让文件堆积
- **立即命名** - 上下文会很快消失
- **有目的地链接** - 只嵌入有价值的内容
- **积极压缩** - 存储会累积
- **大胆删除** - 不是每个截图都重要

## Git 注意事项

### .gitignore 建议
```
*.mp4
*.mov
*.zip
.DS_Store
files_over_10mb/
```

### 对于大文件
- 超过10MB的文件使用 Git LFS
- 考虑外部存储
- 链接到云存储
- 本地保留但添加到gitignore

## 记住

附件是支持你的笔记,而不是替代它们。一个命名良好、组织有序的附件胜过一千张随机截图。使用 Claude Code 的视觉能力来高效地处理和组织可视化内容。
