# 🎯 项目

具有明确成果和截止日期的活跃计划。

## 目的

项目是**有时间限制的工作**，具有：
- 明确的目标和可交付成果
- 明确的开始和结束日期
- 具体的成功标准
- 可衡量的结果

## 什么应该放在这里

### 好的项目示例
- "2024年第一季度营销策略" ✓
- "网站重新设计" ✓
- "书籍写作 - AI与创意" ✓
- "会议演讲 - BRXND NYC" ✓

### 不属于项目（这些应该放在领域中）
- "健康" ✗ (持续进行，没有结束日期)
- "学习" ✗ (持续性活动)
- "团队管理" ✗ (持续性职责)

## 项目结构

每个项目应该包含：
```
项目名称/
├── README.md           # 项目概览和状态
├── 研究/              # 背景资料
├── 每日进展/          # 工作日志
├── 草稿/              # 进行中的工作
├── 资源/              # 支持文档
└── 产出/              # 最终交付成果
```

## Claude Code 工作流程

### 开始一个项目
```
在 01_项目 中创建一个名为 [名称] 的新项目。
使用标准文件夹结构进行设置。
我正在思考模式中 - 帮我定义目标。
```

### 项目研究
```
我正在进行 [项目名称]。
在我的知识库中搜索相关的现有笔记。
与我的其他工作有什么联系？
```

### 每日进展
```
在 [project]/Daily_Progress 中创建今天的进度笔记。
以下是我完成的工作：[摘要]
出现的问题：[列表]
```

### 项目状态
```
查看 [项目文件夹] 中的所有笔记。
当前状态如何？
到目前为止的关键见解是什么？
还剩下什么需要完成？
```

### 项目完成
```
项目 [名称] 已完成。
创建一个回顾，涵盖：
- 目标与成果对比
- 关键学习
- 下次要做不同的事情
然后帮我正确归档。
```

## 项目生命周期

1. **启动**：定义目标、成功标准
2. **研究**：收集相关信息
3. **开发**：每日进展、迭代工作
4. **审查**：定期状态检查
5. **完成**：最终输出、回顾
6. **归档**：移动到 04_归档 并附上摘要

## 成功秘诀

- **从明确的目标开始** - 模糊的项目永远不会结束
- **使用模板** - 一致性有助于 Claude Code 帮助你
- **每日记录进展** - 即使"没有进展"也值得记录
- **每周审查** - 保持项目在正轨上
- **完成或终止** - 不要让项目拖延

## 何时归档

在以下情况下将项目移至 `04_归档/`：
- 所有目标都已完成
- 项目被取消
- 已经闲置30天以上
- 已转变为领域（持续进行）

归档前务必创建摘要笔记！
