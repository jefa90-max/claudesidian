## 项目目标

> [!abstract] 项目定位
> 建设景泰县寺滩乡数字化乡村治理平台，通过**"一端（农户端）+ 一台（管理端）+ 一图（治理大屏）+ 一后台（管理系统）"**的架构，实现乡村治理数字化、便民服务智能化、决策支撑可视化。

### 主要目标
- [ ] 目标 1：完成**数治乡村-农户端**微信小程序开发（44页面），为村民提供随手拍、三务公开、积分超市、便民服务等功能
- [ ] 目标 2：完成**数治乡村-管理端**微信小程序开发（44页面），为网格员/村委提供工单处理、考勤管理、综治管理、信息发布等功能
- [ ] 目标 3：完成**乡村治理一张图**可视化大屏系统开发（6模块30面板），为乡镇领导提供乡情概览、联村共治、宜居乡村、信息惠民、文旅全景、农旅融合等数据展示
- [ ] 目标 4：完成**乡村治理后台管理系统**Web端开发（待规划），为系统管理员提供用户管理、权限配置、数据维护、内容审核等后台管理功能

### 成功标准
- 标准 1：四个子系统全部通过验收测试，功能完整可用
- 标准 2：系统稳定运行，支撑寺滩乡日常治理工作

## 系统架构

```
景泰乡村治理平台
├── 数治乡村-农户端（微信小程序）
│   ├── 首页模块（1页）- 快捷入口、通知、动态
│   ├── 服务模块（11页）- 上工打卡、办事指南、培训、文化云享
│   ├── 三务模块（7页）- 政策公告、阳光村务、曝光台
│   ├── 农旅模块（10页）- 景点、特产、民宿、活动
│   └── 我的模块（15页）- 个人信息、积分管理、设置
│
├── 数治乡村-管理端（微信小程序）
│   ├── 首页模块（6页）- 工作台、待办、快捷操作
│   ├── 综治模块（6页）- 重点人员、网格管理
│   ├── 服务模块（8页）- 用工监管、培训管理
│   ├── 三务模块（10页）- 信息发布、曝光台、预警推送
│   ├── 积分模块（8页）- 积分管理、商品管理、订单统计
│   └── 我的模块（6页）- 个人资料、消息设置
│
├── 乡村治理一张图（可视化大屏）
│   ├── 乡情概览（6面板）- 概览、人口、经济、农业、动态、设施
│   ├── 联村共治（6面板）- 治理架构、综治态势、网格、监控、重点人群
│   ├── 宜居乡村（5面板）- 生态、居住点、人居环境、曝光、应急
│   ├── 信息惠民（5面板）- 便民、服务指南、养老、文化
│   ├── 文旅全景（3面板）- 景点、全景漫游、精品路线
│   └── 农旅融合（5面板）- 农业基地、特产、采摘、民宿、农事
│
└── 乡村治理后台管理系统（Web端）🔜 待规划
    ├── 用户管理 - 账号管理、角色权限、组织架构
    ├── 内容管理 - 公告发布、信息审核、素材管理
    ├── 业务管理 - 工单管理、积分规则、商品上架
    ├── 数据管理 - 基础数据维护、数据导入导出
    └── 系统设置 - 参数配置、日志查看、系统监控
```

## 目标用户

| 用户群体   | 使用系统 | 主要场景                   |
| ------ | ---- | ---------------------- |
| 村民     | 农户端  | 随手拍上报、查看村务公开、积分兑换、培训报名 |
| 网格员/村委 | 管理端  | 处理工单、管理考勤、发布信息、管理积分    |
| 乡镇领导   | 治理大屏 | 查看辖区全貌、数据分析、决策支撑       |
| 系统管理员  | 后台管理 | 用户权限、数据维护、内容审核、系统配置    |
