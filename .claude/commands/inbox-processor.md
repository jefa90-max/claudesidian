# 收件箱处理器

处理 `00_收件箱/` 中的所有内容——日志、网页剪藏、收藏资料、随手笔记、会议记录等。

## 处理流程

### 第 1 步：扫描收件箱

1. 列出 `00_收件箱/` 中所有 `.md` 文件（排除 `README.md`）
2. 如果用户指定了文件名（通过 `$ARGUMENTS`），只处理该文件
3. 如果未指定，列出所有待处理文件，让用户确认处理范围

### 第 2 步：识别内容类型

读取每篇笔记，先判断它属于哪种**来源类型**：

| 类型 | 识别特征 | 处理策略 |
|------|---------|---------|
| 📓 **每日日志** | 文件名为日期格式、有「今日完成/收获/随记」等段落、tags 含「日志/复盘」 | 拆分提炼：有价值内容提取为知识卡片，原始日志归档 |
| 🌐 **网页剪藏** | 有外部 URL、来源标注、大段引用内容、明显是从网页复制的格式 | 整篇移动到对应资源文件夹，必要时补充 frontmatter 和摘要 |
| 📑 **收藏资料** | 课程笔记、教程、方法论总结、别人的经验帖 | 整篇移动到对应资源文件夹，补 tags 和分类 |
| 💡 **随手想法** | 短小、片段化、没有完整结构、属于灵感或念头 | 判断是否值得扩展；如果关联某个领域/项目，合并到对应笔记；否则留在收件箱或归档 |
| 📋 **会议/沟通记录** | 有参会人、讨论要点、待办事项 | 移到对应项目的会议纪要文件夹 |
| 🔧 **项目相关文档** | 需求文档、方案设计、技术调研 | 直接移到对应项目的合适子文件夹 |

### 第 3 步：按类型深度分析

#### 对📓每日日志——拆分提炼

扫描日志内容，识别其中混杂的有价值信息：

| 标记 | 信息类型 | 处理方式 |
|------|---------|---------|
| ⭐ | **决策/转折** — 方向选择、战略判断、重要取舍 | 提炼为独立知识卡片 |
| 📘 | **可复用经验** — 技术踩坑、方法论、流程优化 | 提炼到领域或资源 |
| 📋 | **项目进展** — 具体任务完成、问题、里程碑 | 简要补记到对应项目文件夹 |
| 📝 | **流水账** — 日常事务、已完成待办、过程记录 | 不提炼，跟随日志归档 |
| 💡 | **模糊想法** — 还没想清楚的方向或灵感 | 留在收件箱，下次再看 |

#### 对🌐网页剪藏和📑收藏资料——归类整理

1. 判断内容主题，匹配目标文件夹
2. 检查是否已有 frontmatter，没有则补充：
   ```yaml
   ---
   date: [收藏日期，从文件名或内容推断]
   source: "[原始URL或来源]"
   tags: [根据内容添加]
   type: [剪藏/教程/方法论/课程笔记...]
   ---
   ```
3. 如果内容过长或格式混乱，在顶部添加一句话摘要（`> **摘要**：...`）
4. 检查文件名是否清晰可辨，必要时建议重命名

#### 对💡随手想法——判断去留

- 如果已经与某个领域/项目强相关 → 合并到对应笔记，或在对应位置新建
- 如果仍然模糊 → 留在收件箱
- 如果连续两周以上都还很模糊 → 建议归档（想法不会丢，只是不再占用注意力）

#### 对📋会议记录和🔧项目文档——直接归位

- 识别关联项目，移到项目对应文件夹
- 会议记录 → `XX项目/06-会议纪要/`
- 需求/方案 → `XX项目/02-需求与范围/` 或 `04-方案设计/`

### 第 4 步：输出分析报告

对每篇笔记，输出结构化分析：

```
📄 文件：[文件名]
🏷️ 类型：📓日志 / 🌐剪藏 / 📑资料 / 💡想法 / 📋会议 / 🔧项目文档

处理建议：
  [根据类型展示不同内容]

  --- 日志示例 ---
  ⭐ 放弃小程序转企业AI服务 → 02_领域/副业经营/
  📘 CDP Proxy 端口经验 → 02_领域/技术能力/
  📋 乡村治理任务30.5 → 01_项目/00_政府项目_景泰项目/05-执行推进/
  📝 其余流水账 → 跟随归档

  --- 剪藏/资料示例 ---
  整篇移动 → 03_资源/AI编程教程/
  补充 frontmatter + 摘要

  --- 想法示例 ---
  合并到 → 02_领域/副业经营/垂直小号定位.md
  或：留在收件箱（仍不清晰）

建议动作：提炼 N 条 + 移动 / 直接移动 / 直接归档 / 保留
```

### 第 5 步：等待用户确认

展示分析报告，等用户确认后再执行。用户可能：
- 同意全部建议
- 调整某些内容的去向
- 跳过某些笔记

### 第 6 步：执行分发

用户确认后，按类型执行：

#### 6a. 日志提炼——创建知识卡片（⭐ 和 📘）

```markdown
---
date: [提炼日期]
source: "[[04_归档/日志/原始日志文件名]]"
tags: [根据内容添加]
---

# [标题]

## 背景
[这个认知/决策/经验产生的场景]

## 核心内容
[具体的知识点、决策、方法论]

## 为什么重要
[对后续的影响和意义]

## 相关链接
- [[原始日志链接]]
```

#### 6b. 日志中的项目进展（📋）

在对应项目的执行推进文件夹中，追加简要记录。

#### 6c. 剪藏/资料——补充 frontmatter 后移动

补充缺失的 frontmatter 字段，然后移到目标文件夹。

#### 6d. 想法——合并或保留

如果合并，追加到目标笔记末尾并标注来源日期；如果保留，不动。

#### 6e. 会议/项目文档——直接移动

移到对应项目子文件夹。

#### 6f. 归档

- 日志：添加 `processed: true` 到 frontmatter → 移到 `04_归档/日志/`
- 其他已处理的笔记：如果整篇移走了，收件箱里就没有残留

### 第 7 步：输出处理报告

```
✅ 收件箱处理完成

已提炼（从日志中）：
  - [[新笔记1]] → 02_领域/副业经营/
  - [[新笔记2]] → 02_领域/技术能力/

已移动（剪藏/资料）：
  - 文章标题.md → 03_资源/AI编程教程/
  - 课程笔记.md → 03_资源/投资课程笔记/

已补记（项目进展）：
  - 景泰项目 → 01_项目/00_政府项目_景泰项目/05-执行推进/

已归档（日志）：
  - 2026-03-13.md → 04_归档/日志/

仍在收件箱：
  - 某个想法.md（💡 仍不清晰，下次再看）
```

## 分发目标速查

### 项目（`01_项目/`）

| 内容主题 | 目标位置 |
|---------|---------|
| 景泰/乡村治理项目 | `01_项目/00_政府项目_景泰项目/` 下对应子文件夹 |
| 油管英语网站 | `01_项目/01_副业项目_油管英语网站/` |
| 公众号垂直小号 | `01_项目/02_副业项目_公众号垂直小号/` |
| 小红书店铺 | `01_项目/03_副业项目_小红书店铺/` |
| X 账号增长 | `01_项目/04_副业项目_X账号增长/` |

### 领域（`02_领域/`）

| 内容主题 | 目标位置 |
|---------|---------|
| 副业方向/战略思考 | `02_领域/副业经营/` |
| 技术经验/踩坑 | `02_领域/技术能力/` |
| 投资理财 | `02_领域/投资理财/` |
| 儿童教育 | `02_领域/儿童教育/` |

### 资源（`03_资源/`）

| 内容主题 | 目标位置 |
|---------|---------|
| AI 应用/提示词 | `03_资源/AI应用与提示词/` |
| AI 编程教程 | `03_资源/AI编程教程/` |
| 自媒体运营方法 | `03_资源/自媒体运营方法论/` |
| 投资课程笔记 | `03_资源/投资课程笔记/` |

### 如果不匹配以上任何分类

- 看看是否需要在 `02_领域/` 或 `03_资源/` 下新建子文件夹
- 问用户确认，不要自作主张新建

## 注意事项

- 用中文写所有内容
- 知识卡片的 `source` 字段使用归档后的路径
- 如果目标文件夹下缺少子文件夹，先创建再放入
- 项目进展的补记要简洁，一两句话即可
- 剪藏/资料移动时保留原始内容完整性，只补充 frontmatter 和摘要
- 遇到不确定的归属，宁可问用户，不要猜
- 完整流程参考：`06_元数据/参考/收件箱处理流程.md`
