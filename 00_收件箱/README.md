## 目的

收件箱是一个**临时**存放区，用于:
- 快速捕捉内容而无需担心如何组织
- 每日笔记和日记
- 网页剪藏和有趣的发现
- 会议记录(处理之前)
- 随机的想法和创意

## 如何使用

### 日常工作流程
1. **先捕捉,后整理** - 不要为了分类而减慢速度
2. **使用每日笔记** - 每天一篇笔记用于意识流记录
3. **随手保存链接和引用** - 现在保存,稍后处理
4. **不要让它堆积** - 每周处理一次

### 每周处理流程
每周审查你的收件箱并:
- 将项目相关笔记移至 `01_项目/`
- 将持续关注的主题移至 `02_领域/`
- 将参考资料移至 `03_资源/`
- 将已完成的项目归档至 `04_归档/`
- 删除不再相关的内容

## Claude Code 命令

### 快速捕捉
```
在 00_收件箱 中创建一个新笔记,标题为 [标题]
内容是这些想法: [内容]
```

### 处理收件箱
```
审查 00_收件箱 中的所有笔记。
根据 PARA 方法帮我决定每篇笔记应该放在哪里。
```

### 发现联系
```
查看我收件箱中的最新笔记。
你看到了什么模式或主题?
这些内容可能与哪些现有笔记相关?
```

## 小贴士

- **不要追求完美** - 收件箱本来就应该是杂乱的
- **使用描述性文件名** - 让处理变得更容易
- **给笔记加上日期** - YYYY-MM-DD 格式有助于排序
- **自由地添加链接** - 即使是指向尚不存在的笔记
- **定期审查** - 不要让项目停留超过 2 周

## 文件命名

建议格式:
- 每日笔记: `2024-03-15.md`
- 快速捕捉: `2024-03-15 - 与团队会议.md`
- 网页剪藏: `2024-03-15 - 文章标题 - 来源.md`
- 想法: `想法 - 简短描述.md`

记住: 收件箱是想法开始的地方,而不是它们永久居住的地方。
