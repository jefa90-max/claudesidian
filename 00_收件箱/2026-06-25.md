## ✍️ 过程记录
### 上午
- VPS 代理排障：Clash Verge 无法访问外网，VPS 端 s-ui 日志持续出现 `REALITY: processed invalid connection`，判断不是 VPS 整体网络断开，而是 VLESS Reality 入站握手失败。
- 排查过程：确认 s-ui 服务仍在运行，`443 / 2095 / 2096` 等端口有监听；`th.bing.com:443` 作为 Reality 伪装目标可从 VPS 访问；Windows 与 VPS 时间均已同步；重启 s-ui 后 Reality 仍立即报握手失败。
- 临时解决方案：在 Clash Verge 中切换到 `TUIC-443` 节点后恢复可用，说明 s-ui/代理服务整体正常，当前主要问题集中在 `VLESS Reality`，短期继续使用 TUIC 节点。
- 安全加固：发现 VPS 登录提示中有非本人产生的多次 SSH 登录失败记录，判断为公网扫描/爆破尝试；开始将 SSH 改为 key 登录并禁用密码登录，同时安装并启用 fail2ban 保护 `sshd`。
- fail2ban 安装处理：最初 `dnf install fail2ban` 找不到包，原因是 EPEL 主源未正确启用、只启用了 `epel-cisco-openh264`；启用 EPEL 主源后安装成功，并确认 `fail2ban` 服务处于 `active (running)`，`sshd` jail 已生效。

## 📚 今日收获
<!-- 学到了什么？有什么新认知？ -->
- `REALITY: processed invalid connection` 更偏向 Reality 握手认证/协议兼容问题，不代表 VPS 或 s-ui 整体不可用；通过切换 TUIC 节点可以快速验证代理服务是否整体正常。
- OpenCloudOS/CentOS 系安装 fail2ban 时，如果只启用了 `epel-cisco-openh264` 而没有启用 EPEL 主源，会出现找不到 `fail2ban` 包的问题。
- VPS 出现非本人 SSH 登录失败记录时，应优先做 SSH key 登录、禁用密码登录、启用 fail2ban，而不是只改代理配置。
