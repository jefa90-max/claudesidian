# 📥 收件箱处理流程

## 核心原则

```
收件箱 → 提炼 → 分发 → 归档
```

收件箱是**临时中转站**，不是日记本。每篇日志最终要完成三件事：
1. 有价值的内容 **提炼** 成独立笔记
2. 提炼后的笔记 **分发** 到对应位置
3. 原始日志 **归档** 到 `04_归档/日志/`

## 处理节奏

| 频率 | 动作 |
|------|------|
| **每天写** | 在 `00_收件箱/` 用"每日工作一体化模板"记录 |
| **每周清** | 周末花 15-30 分钟处理本周的收件箱笔记 |
| **每月审** | 月初快速扫一眼 `04_归档/日志/`，确认没漏掉重要内容 |

## 每周处理步骤

### 第 1 步：扫描（5 分钟）

快速浏览本周所有收件箱笔记，在每篇底部的「🧊 待提炼」区标记有价值的内容：

```markdown
## 🧊 待提炼
- [ ] CDP Proxy 选端口的经验 → 02_领域/技术能力
- [ ] 放弃小程序转企业AI服务的决策 → 02_领域/副业经营
- [ ] 去AI味的文案方法 → 03_资源/自媒体运营方法论
```

### 第 2 步：判断——值不值得提炼？

不是所有内容都需要变成独立笔记。用这个标准快速筛选：

| 类型 | 处理方式 | 举例 |
|------|---------|------|
| ⭐ **决策/转折** | 必须提炼为独立笔记 | "放弃小程序方向"、"职场+AI路线确认" |
| 📘 **可复用经验** | 提炼到领域或资源 | "CDP端口选择"、"去AI味方法" |
| 📋 **项目进展** | 简要记入项目的执行推进 | "完成任务30.5"、"720控件方案" |
| 📝 **流水账** | 不提炼，直接归档 | "取了羽绒服"、"启动了后端服务" |
| 💡 **模糊想法** | 先留在收件箱，下周再看 | "这个可以做但还没想清楚" |

### 第 3 步：提炼（10-20 分钟）

对标记为需要提炼的内容：

1. 用「知识卡片模板」创建新笔记
2. 放到对应文件夹
3. 在新笔记的 `source` 字段回链原始日志
4. 在原始日志中勾选已提炼的项目 `- [x]`

### 第 4 步：归档（2 分钟）

1. 将 frontmatter 中 `processed: false` 改为 `processed: true`
2. 把日志文件移到 `04_归档/日志/`

## 分发目标速查

| 内容类型 | 目标位置 | 什么时候用 |
|---------|---------|-----------|
| 景泰/乡村治理项目进展 | `01_项目/00_政府项目_景泰项目/05-执行推进/` | 有具体进展、问题、决策 |
| 油管英语网站进展 | `01_项目/01_副业项目_油管英语网站/每日进展/` | 功能完成、上线、问题 |
| 小红书/公众号进展 | `01_项目/02_ 或 03_副业项目.../` | 内容发布、数据、优化 |
| 副业方向/战略思考 | `02_领域/副业经营/` | 方向选择、商业判断 |
| 技术经验/踩坑记录 | `02_领域/技术能力/` | 可复用的技术知识 |
| 投资/理财决策 | `02_领域/投资理财/` | 买卖操作、策略调整 |
| 方法论/教程 | `03_资源/对应分类/` | 通用参考价值的内容 |
| 纯历史记录 | `04_归档/日志/` | 流水账、已完成待办 |

## 常见问题

**Q: 一篇日志里有 5 个不同主题，要拆成 5 篇笔记吗？**
A: 不一定。只提炼 ⭐决策/转折 和 📘可复用经验。项目进展只需在项目笔记里补一行记录即可，不用单独建笔记。

**Q: 有些想法还没想清楚，不知道放哪？**
A: 留在收件箱，下周再看。如果连续两周都"还没想清楚"，说明这个想法可能本身就不重要，直接归档。

**Q: 日志处理完后还能找到吗？**
A: 能。归档日志完整保留在 `04_归档/日志/`，用 Obsidian 搜索即可。提炼出的知识卡片会通过 `source` 字段回链到原始日志。
