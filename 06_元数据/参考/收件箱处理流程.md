# 📥 收件箱处理流程

## 核心原则

```
收件箱 → 识别类型 → 分析 → 分发 → 归档
```

收件箱是**临时中转站**。不管是日志、网页剪藏、收藏资料还是随手笔记，最终都要被处理到它该去的地方。

## 收件箱里会有什么？

| 类型 | 举例 | 处理方式 |
|------|------|---------|
| 📓 **每日日志** | 工作记录、复盘 | 拆分提炼有价值内容 → 知识卡片 + 项目补记，原始日志归档 |
| 🌐 **网页剪藏** | 文章、教程、帖子 | 补 frontmatter → 整篇移到对应资源文件夹 |
| 📑 **收藏资料** | 课程笔记、方法论 | 补 frontmatter → 整篇移到对应资源文件夹 |
| 💡 **随手想法** | 灵感、片段 | 判断去留：合并到已有笔记 / 留在收件箱 / 归档 |
| 📋 **会议记录** | 沟通纪要、待办 | 直接移到对应项目的会议纪要文件夹 |
| 🔧 **项目文档** | 需求、方案、调研 | 直接移到对应项目的合适子文件夹 |

## 处理节奏

| 频率 | 动作 |
|------|------|
| **随时** | 往 `00_收件箱/` 扔任何东西，不用管分类 |
| **每周清** | 周末花 15-30 分钟处理本周的收件箱内容 |
| **每月审** | 月初快速扫一眼，清理超过两周的「💡模糊想法」 |

## 每周处理步骤

### 第 1 步：扫描（3 分钟）

快速浏览收件箱所有笔记，给每篇在心里打个标签：这是日志？剪藏？资料？想法？

### 第 2 步：按类型分别处理

#### 📓 日志类——拆分提炼

在日志底部的「🧊 待提炼」区标记有价值内容：

```markdown
## 🧊 待提炼
- [ ] CDP Proxy 选端口的经验 → 02_领域/技术能力
- [ ] 放弃小程序转企业AI服务 → 02_领域/副业经营
- [ ] 去AI味的文案方法 → 03_资源/自媒体运营方法论
```

用这个标准判断值不值得提炼：

| 类型 | 处理方式 | 举例 |
|------|---------|------|
| ⭐ **决策/转折** | 必须提炼为独立笔记 | "放弃小程序方向"、"职场+AI路线确认" |
| 📘 **可复用经验** | 提炼到领域或资源 | "CDP端口选择"、"去AI味方法" |
| 📋 **项目进展** | 简要记入项目执行推进 | "完成任务30.5" |
| 📝 **流水账** | 不提炼，直接归档 | "取了羽绒服" |
| 💡 **模糊想法** | 留在收件箱，下周再看 | "这个可以做但还没想清楚" |

提炼后：`processed: true` → 移到 `04_归档/日志/`

#### 🌐📑 剪藏/资料——补充后移动

1. 检查有没有 frontmatter，没有就补上（date、source、tags、type）
2. 内容太长就加一句摘要
3. 文件名不清晰就重命名
4. 移到对应的 `03_资源/` 子文件夹

#### 💡 随手想法——判断去留

- 已经清晰 + 关联某个领域 → 合并或新建到对应位置
- 还很模糊 → 留在收件箱
- 超过两周仍然模糊 → 归档（不会丢，只是不占注意力）

#### 📋🔧 会议/项目文档——直接归位

- 会议记录 → `01_项目/XX项目/06-会议纪要/`
- 需求/方案 → `01_项目/XX项目/02-需求与范围/` 或 `04-方案设计/`

### 第 3 步：检查清场（2 分钟）

收件箱里应该只剩下：
- 本周新写的、还没到处理时间的日志
- 标记为💡的模糊想法（等下周再看）
- 其他都应该已经被移走或归档

## 用 `/inbox-processor` 自动处理

最快的方式是直接用命令：

```
/inbox-processor                    ← 处理全部收件箱
/inbox-processor 2026-03-13.md      ← 只处理指定文件
```

命令会自动识别内容类型、分析、生成建议报告，确认后再执行。

## 常见问题

**Q: 一篇日志里有 5 个不同主题，要拆成 5 篇笔记吗？**
A: 不一定。只提炼 ⭐决策/转折 和 📘可复用经验。项目进展只需在项目笔记里补一行即可。

**Q: 网页剪藏需要重新整理格式吗？**
A: 不需要大改。补上 frontmatter 和摘要就行，保留原始内容完整性。

**Q: 有些想法还没想清楚，不知道放哪？**
A: 留在收件箱。超过两周都没想清楚 → 直接归档，说明当下不重要。

**Q: 处理完后还能找到原始内容吗？**
A: 能。日志归档在 `04_归档/日志/`，提炼的知识卡片通过 `source` 字段回链。剪藏/资料整篇移走，内容完整保留。
