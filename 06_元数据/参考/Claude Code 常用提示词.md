# Claude Code 常用提示词

在你的知识库中使用 Claude Code 的实用提示词集合。

## 开始工作

### 开始会话
```
我今天开始工作了。
你能回顾一下我昨天在做什么，
帮我从上次中断的地方继续吗？
```

### 设置模式
```
我现在处于思考模式，而不是写作模式。
请通过提问和搜索相关笔记，
帮我探索 [主题]。
```

## 研究与综合

### 发现关联
```
搜索我的知识库中与 [主题] 相关的所有内容。
你发现了什么规律或关联？
```

### 项目综合
```
查看 [项目文件夹] 中的所有笔记。
对关键主题、见解和待解决问题进行综合总结。
```

### 周回顾
```
查看本周创建的所有笔记。
主要主题是什么？
不同项目之间存在哪些关联？
```

## 整理组织

### 处理收件箱
```
查看 00_Inbox 中的项目。
根据 PARA 方法建议每个项目应该移动到哪里。
哪些项目可以合并或链接？
```

### 查找孤立笔记
```
找出没有链接到任何其他笔记的笔记。
建议可能的关联。
```

### 清理附件
```
查看 05_Attachments 中的文件。
哪些文件没有在任何笔记中被引用？
哪些文件可以更好地命名？
```

## 写作与创作

### 切换到写作模式
```
我准备从思考模式转入写作模式。
基于我们在 [项目] 中的研究，
帮我为 [成果物] 创建一个大纲。
```

### 改进草稿
```
查看 [文档]。
不要重写它，但请给我具体反馈：
- 结构和流程
- 逻辑或论据的漏洞
- 需要澄清的地方
```

## 学习与发展

### 探索新主题
```
我想学习 [主题]。
首先搜索我的知识库中已有的相关知识。
然后帮我确定需要研究什么。
```

### 构建论点
```
我想论证 [论点]。
搜索我的笔记寻找支持证据。
我应该应对哪些反驳意见？
```

## 项目管理

### 项目状态
```
查看 [文件夹] 中的项目。
当前状态是什么？
需要采取哪些下一步行动？
```

### 创建复盘
```
[项目] 现已完成。
查看所有笔记并创建复盘报告，涵盖：
- 完成了什么
- 关键收获
- 下次应该改进什么
```

## 日常操作

### 晨间回顾
```
早上好。告诉我：
- 昨天修改过的笔记
- 待办任务或问题
- 今天我应该专注于什么？
```

### 一天结束总结
```
一天结束回顾：
- 今天我完成了什么？
- 产生了哪些问题或想法？
- 明天应该优先处理什么？
```

## 进阶技巧

### 跨项目分析
```
比较 [项目A] 和 [项目B] 的见解。
两者之间存在什么规律？
它们能从彼此学到什么？
```

### 知识缺口
```
分析我关于 [主题] 的笔记。
我遗漏了哪些方面？
我还没有问过哪些问题？
```

### 想法发展
```
我有这个初步想法：[想法]
在我的知识库中搜索相关概念。
帮我把它发展成更具体的东西。
```

## 有效提示词的技巧

1. **明确指定模式**（思考 vs 写作）
2. **引用特定文件夹**（如相关的话）
3. **要求提问**，而不只是答案
4. **要求综合**，而不只是搜索
5. **自由迭代** - 进行对话

## 记住

- Claude Code 可以访问你的整个知识库
- 它可以创建、编辑和组织文件
- 把它当作思考伙伴，而不只是工具
- 最好的提示词源于你的具体需求
