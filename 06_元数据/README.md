# ⚙️ 元数据

Vault 配置、文档和组织工具。

## 目的

元数据文件夹包含：
- 关于 vault 的文档
- 用于创建一致笔记的模板
- 参考指南和操作指南
- Agent 配置
- 工作流文档

## 结构

```
06_元数据/
├── 参考/            # 指南和文档
├── 模板/            # 笔记模板
├── 代理/            # Claude Code agent 配置
├── 工作流/          # 文档化的流程
└── 归档/            # 旧配置
```

## 使用模板

### 手动方式
1. 复制模板内容
2. 创建新笔记
3. 粘贴并填写

### 使用 Claude Code
```
使用项目模板创建一个新项目。
命名为 [项目名称] 并放在 01_项目 中。
```

## 创建自定义 Agents

将 agent 指令保存为 markdown 文件：

```markdown
# Agent: [名称]

你是一个 [角色描述]。

## 核心行为
- 行为 1
- 行为 2

## 工作流
1. 步骤 1
2. 步骤 2

## 约束
- 不要做 X
- 始终做 Y
```

然后在 Claude Code 中引用：
```
使用 06_元数据/代理/[agent].md 中的指令
帮我完成 [任务]。
```

## Claude Code 提示词

### 模板使用
```
向我展示 06_元数据/模板 中的可用模板。
使用适当的模板创建一个新的 [类型] 笔记。
```

### 文档
```
在 06_元数据/参考 中查看 [主题] 的文档。
根据我们刚学到的内容更新指南。
```

### 工作流执行
```
从 06_元数据/工作流 运行每周回顾工作流。
引导我完成每个步骤。
```

## 维护

### 定期更新
- 根据使用情况更新模板
- 记录新出现的工作流
- 归档过时的配置
- 保持参考文档最新

### 版本控制
- 跟踪工作流的变更
- 记录为什么进行更改
- 保留旧版本的存档
- 标注主要更新日期

## 最佳实践

- **边做边记录** - 趁记忆犹新时捕捉工作流
- **迭代模板** - 根据使用情况改进
- **分享配置** - 对你有用的可能也能帮助他人
- **保持简单** - 复杂的系统容易崩溃
- **标注日期** - 上下文很重要

## 记住

元数据是你的 vault 的操作系统。良好的元数据意味着一致的结构、可重复的工作流和可扩展的增长。这是你不仅记录知识，还记录工作方式的地方。
