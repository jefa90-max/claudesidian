# 🗄️ 归档

为未来参考保留的非活跃项目。

## 目的

归档存储:

- 已完成的项目及其成果
- 不再维护的非活跃领域
- 历史参考的旧笔记
- 已弃用的资源
- 过去的实验和迭代

## 什么内容应该放在这里

### 来自项目

- 包含最终交付成果的已完成项目
- 记录经验教训的已取消项目
- 非活跃超过30天的项目

### 来自领域

- 不再与你的生活相关的领域
- 你已移交的职责
- 你不再担任的角色

### 来自资源

- 过时的信息(但具有历史趣味)
- 被取代的框架或方法
- 已演化思想的旧版本

### 来自收件箱

- 已处理但不再需要的项目
- 旧的每日笔记(提取后)
- 没有持久价值的随机记录

## 组织结构

```
04_归档/
├── 项目_2024/        # 按年份分类的已完成项目
├── 项目_2023/
├── 旧领域/           # 已停止的领域
├── 每日笔记/         # 旧的每日记录
├── 想法/             # 未发展的想法
└── 杂项/             # 其他所有内容
```

## 归档流程

### 归档项目之前

1. 创建完成总结
2. 提取可复用的见解到资源
3. 记录经验教训
4. 更新任何相关领域
5. 保持结构完整地移动整个文件夹

### 完成总结示例

```markdown
# 项目: [名称] - 完成总结

**持续时间**: 开始日期 - 结束日期
**状态**: 已完成/已取消/已暂停

## 目标

- 原始目标 1 ✓
- 原始目标 2 ✓
- 原始目标 3 ✗

## 关键成果

- 交付了什么
- 产生了什么影响
- 创造了什么价值

## 经验教训

- 什么做得好
- 什么没做好
- 下次要如何改进

## 可复用资产

- 创建的模板: [[链接]]
- 开发的流程: [[链接]]
- 获得的见解: [[链接]]

## 相关笔记

- 延续于: [[领域名称]]
- 另见: [[相关项目]]
```

## 归档哲学

### 它不是墓地

- 归档保存组织记忆
- 旧项目启发新项目
- 模式随时间浮现
- 想法可以复活

### 它不是垃圾场

- 有思考地归档
- 保持一定的组织性
- 保持总结的可访问性
- 删除真正的垃圾

## Claude Code 提示

### 历史分析

```
查看我的已归档项目。
我倾向于开始但没有完成什么类型的事情?
什么主题反复出现?
```

### 知识挖掘

```
在归档中搜索任何提到 [概念] 的内容。
我的思维是如何演变的?
```

### 模式识别

```
分析我的项目完成率。
什么因素与成功相关?
什么模式预示失败?
```

## 维护

### 每季度

- 回顾最近的添加
- 确保总结存在
- 检查可复活的候选项目

### 每年

- 重大归档清理
- 删除真正没用的内容
- 提取任何遗漏的见解
- 根据需要重新组织

## 技巧

- **给所有内容标注日期** - 未来的你会感谢你
- **总是进行总结** - 背景消退得很快
- **自由地链接** - 连接可以在归档后存活
- **经常搜索** - 归档是用来使用的
- **无畏地删除** - 不是所有东西都需要保留

## 记住

归档是你的组织记忆。它不是关于保留所有东西,而是关于保存可能启发未来工作的内容。使用 Claude Code 帮助你看到跨时间的模式,并从经验中提取智慧。
